A Kick in the Class!

(Or get a kick out of class)

May 31, 2015

In Ruby, a Class is what is called a first-class object. Other examples of first class objects are Strings and Integers. Inside of a class, you can define objects and methods. But, here's the kicker... YOU CAN DEFINE YOUR OWN CLASS!. That means you can create a class with objects and methods specific to that class. Let's take a look.

Let's say we are writing a new song and want to store some basic information about it. Firstly, we start by defining the class. 

class Song
  #code will go here
end

That's great, but now what? We need to create a method called initialize. This method will run for every new instance of the class. For our initialize we will define some variables that will contain information about the song. But not just any variables, instance variables! Instance variables are unique to each instance of the class. Makes sense, right? To define these variables our initialize method will take a few arguments, and the instance variables will be assigned to these objects.

class Song
  def initialize(title, genre, key, time, tempo)
    @title = title
    @genre = genre
    @key = key
    @time = time
    @tempo = tempo
  end
end

Those variables look a bit different than the ones we're used to. The only difference in syntax is that instance variables have "@" in front of them.

Now, let's define a method for this class that takes all of the information about your song and prints it to the console

class Song
  def initialize(title, genre, key, time, tempo)
    @title = title
    @genre = genre
    @key = key
    @time = time
    @tempo = tempo
  end

  def song_info
    p "You are writing a #{@genre} song called #{@title}!"
    p "It is in the key of #{@key} in #{@time} at #{@tempo}."
  end
end

Pretty straight-forward. Now we have a method that will tell us all about our song when we call it on an instance of the class. First, we need to call a new instance of the class. We need to make sure that the class is capitalized and that we have the correct number of arguments.

new_song = Song.new("Ruby Bogonias", "Psychadelic Rock", "C# minor", "7/8", "160 BPM")

We have a new instance of the class now, so we can call our song_info method on this instance and see what kind of song we have!

new_song = Song.new("Ruby Bogonias", "Psychadelic Rock", "C# minor", "7/8", "160 BPM")
new_song.song_info

"You are writing a Psychadelic Rock song called Ruby Bogonias!"
"It is in the key of C# minor in 7/8 at 160 BPM."

Awesome! But what if we only want to see one aspect of our song instead of everything? We can just print the variable, right? Wrong! It's an instance variable, so it is unique to this instance of the method, remember? Ruby has a really handy way of sharing instance variables called attributes. If we attribute the variables, then we can use each variable as a method on the instance of its class and look at it by itself. We need to add a little something to the class for this.

class Song

  def initialize(title, genre, key, time, tempo)
    @title = title
    @genre = genre
    @key = key
    @time = time
    @tempo = tempo
  end

attr_reader :title

  def song_info
    p "You are writing a #{@genre} song called #{@title}!"
    p "It is in the key of #{@key} in #{@time} at #{@tempo}."
  end

end

Now, we can see only the title of the song if we want!

new_song = Song.new("Ruby Bogonias", "Psychadelic Rock", "C# minor", "7/8", "160 BPM")
p new_song.title

"Ruby Bogonias"

We got the title! We can attribute all of the instance variables, and we dont even need to to each one individually. We can do it all at once like this:

attr_reader :title, :genre, :key, :time, :tempo

There is a bunch more stuff that you can do with classes in Ruby, but I hope you got a basic understanding from my little kick in the class!
