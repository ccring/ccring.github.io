Arrays and Hashes

What's the difference between [] and {}?

May 17, 2015

Arrays are two slightly different, but equally important methods of storing information in Ruby.

An array is a list of values that are indexed by integer values. This first value is at index 0, the second is at 1, the third is at 2, etc. If you would like to count from the last value of the array, the index should be a negative number (-1 is the last value). Here is what an array looks like: 

array = [1, "two", 3.0, [4, "five"]]

As you can see, you can put almost any type of element you would like in an array. It can be an integer, a string, or even another array! There are multitudinous methods that you can use to iterate over an array, so you can do almost whatever you want with the information contained. You can take a look at the Ruby-doc to see everything that you can do with an array.

A hash is a delicious dish usually made of meat and potatoes with some spices thrown in. Add a fried egg on top to make it a delicious breakfast

...But also a hash is another powerful type of information storage in Ruby. As opposed to an array, the information in a hash is not ordered by an integer index, but rather a key assigned to each value contained within the hash. The value can be almost anything, similar to the information in an array. Here is what a hash looks like: 

hash = { "name" => "Connor"
"age" => 23
"favorite food" => "Fried Chicken" }

The values can be called by their respective keys, and just like an array you can iterate over the hash and do whatever you want with it. Again, you can take a look at the Ruby-doc to see all the methods that you can call on a hash.
