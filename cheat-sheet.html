Our Fabulous Cheatsheet

Hashes

A Hash is a dictionary-like collection of unique keys and their values. Also called associative arrays, they are similar to Arrays, but where an Array uses integers as its index, a Hash allows you to use any object type.

  • Hashes enumerate their values in the order that the corresponding keys were inserted.
  • Hashes are best used when you would like your data to be easily accessible by calling a name or symbol that represents it. In the context of hashes this is called a key-value pair.
  • A Hash can be easily created by using its implicit form:

    • age = { "Connor" => 23, "Andrew" => 25, "Steve" => 37 }

  • A hash can also be created using the ':' to create symbol-value pairs

    • hash = {city_popuation: 400000, city_size: 42}

Commonly Associated Methods

Hash#each {|key,value| block } Hash#flatten

Arrays

Arrays are ordered, integer-indexed collections of any object.

  • Array indexing starts at 0, as in C or Java. A negative index is assumed to be relative to the end of the array---that is, an index of -1 indicates the last element of the array, -2 is the next to last element in the array, and so on.
  • A new array can be created by using the literal constructor [] and may containt any type of object within it.

    • ary = [1, "two", 3.0]

Commonly Associated Methods

Array#map {|element| block} Array#include? Array#each(.with_index)

Strings

A String object holds and manipulates an arbitrary sequence of bytes, typically representing characters. String objects may be created using String::new or as literals.

  • A string is constructed within single or double quotations
  • A single string is similarly indexed like an array, where string[0] will yield the first character, and string[-1] yields the last character

Commonly Associated Methods

gsub! split insert

Integers

Integer is the basis for the two concrete classes that hold whole numbers, Bignum and Fixnum.

  • Outside of the Bignum class, integers can either be fixed (i.e. 3, 5, 49) or floats (1.4,3.242, 4.0001)
  • Bignums are objects that are created automatically when integer calculations would otherwise overflow a Fixnum.

Commonly Associated Methods

Standard mathmetical Operators: + - * / Modulus % Floor
